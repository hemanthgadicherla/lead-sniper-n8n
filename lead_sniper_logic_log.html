1. Workflow Overview

This workflow automatically monitors new stargazers on a GitHub repository, enriches each user's profile via the GitHub REST API, filters for high-value leads based on follower count and public repo count, uses an LLM to generate a personalized sales pitch, and posts a formatted Block Kit alert to Slack.

2. GitHub API Rate Limit Strategy

GitHub's REST API enforces strict rate limits. This section explains exactly how the workflow handles them.

ScenarioRate Limit
Unauthenticated requests60 requests / hour
Authenticated (PAT) requests5,000 requests / hour
Search API (authenticated)30 requests / minute
Stargazers per API pageUp to 100 items per call

2.1 Authentication Header

Every HTTP Request node sends an Authorization header with a GitHub Personal Access Token (PAT) stored as an n8n httpHeaderAuth credential. This upgrades the anonymous 60 req/hr limit to 5,000 req/hr.

  • Header: Authorization: Bearer <YOUR_GITHUB_PAT>
  • Also sent: Accept: application/vnd.github+json
  • Also sent: X-GitHub-Api-Version: 2022-11-28

The PAT only needs the public_repo read scope — no write permissions required.

2.2 Per-User Delay (Wait Node)

Each stargazer is processed one at a time via a SplitInBatches node (batch size = 1). A Wait node introduces a 1-second pause between each /users/{login} enrichment call.

  • Maximum throughput: ~60 enrichment calls / minute
  • Well within the 5,000/hour authenticated limit
  • Prevents burst spikes that trigger GitHub's secondary rate limits
GitHub can apply secondary rate limits (HTTP 429) even within the primary quota if requests arrive too fast in a short window. The 1-second wait eliminates this risk entirely.

2.3 Rate Limit Response Headers

GitHub returns these headers on every response. Enable Full Response on the HTTP Request node to access them:

  • X-RateLimit-Remaining — requests left in current window
  • X-RateLimit-Reset — Unix timestamp when the quota resets
  • Retry-After — seconds to wait if a 429 is returned

Recommended enhancement: add a Code node after each GitHub call that checks X-RateLimit-Remaining. If the value drops below 100, trigger an additional Wait node (60 seconds) before continuing.

2.4 Polling Frequency

The Schedule Trigger fires every 15 minutes. Each poll fetches the stargazers list with per_page=30, processing at most 30 new stars per cycle.

  • Maximum: 4 stargazer list calls/hour per repo
  • Maximum: 120 enrichment calls/hour at full 30-per-cycle throughput
  • This is only 2.4% of the authenticated rate limit — extremely conservative

3. Filter Logic

The IF node uses an ANY combinator (logical OR) with two conditions:

Conditionn8n ExpressionMeaning
High follower count{{ $json.followers }} > 100Influential developer or public figure
Active contributor{{ $json.public_repos }} > 50Prolific builder, likely technically sophisticated

If either condition is true, the user proceeds to the AI pitch node. If both are false, the false output branch is left unconnected — n8n silently terminates the execution branch with no error or noise.

4. AI Pitch Node

The OpenAI node (model: gpt-4o-mini) receives a constrained prompt injecting the user's name, company, bio, location, public_repos, and followers. It is instructed to output exactly one sentence under 25 words.

  • Model chosen: gpt-4o-mini — cost-efficient ($0.00015/1K input tokens)
  • Estimated cost: ~$0.0001 per lead → under $1/month for 10,000 leads
  • The bio field is the most signal-rich — it often reveals tech stack, interests, and seniority

5. Slack Output Format

The Build Slack Payload Code node assembles a Slack Block Kit message with these sections:

  • Header block: "🎯 New High-Value Lead"
  • Fields section: Name (hyperlinked to GitHub), Company, Followers, Public Repos, Location, Account Created
  • Bio section: Full bio text
  • AI Pitch section: Generated pitch in a blockquote
  • Divider: Visual separator for multi-lead feeds

6. Credentials Required

CredentialHow to Create
GitHub PAT (httpHeaderAuth)GitHub → Settings → Developer settings → Personal access tokens → Fine-grained. Scope: Contents: read-only on public repos.

Slack Incoming Webhook URLapi.slack.com → Your Apps → Incoming Webhooks → Add new webhook. Paste the URL directly into the "Post to Slack" HTTP node.

7. Import Instructions

  • In n8n, click + New Workflow → Import from file → upload lead_sniper_workflow.json
  • Open the Get Stargazers node and update the URL to your target repo (default: tiangolo/fastapi)
  • Assign your GitHub PAT credential to both HTTP Request nodes
  • Assign your OpenAI credential to the Generate Sales Pitch node
  • Paste your Slack Webhook URL into the Post to Slack node
  • Click Execute Workflow once manually (uses pinned test data — no live API calls)
  • Click Activate to start the 15-minute polling schedule
